# /产品叙事策略

当用户需要为产品构建叙事策略时，按以下框架逐层分析并输出方案。

## 框架总览

```
轴（底层机制）→ 模态（载体）→ 操作维度（策略）→ 媒介适配（执行）
                                    ↑
                              FAB（内容翻译器，贯穿轴与操作维度）
```

四层递进，三根轴贯穿所有层，FAB作为翻译器嵌入轴与操作维度之间。

---

## 一、轴（叙事的底层机制）

三根轴贯穿所有层，是框架的地基。

### 轴1. 意义绑定
将产品从功能物件转化为意义载体。绑定方向三选一或组合：
- **绑身份**：用这个产品的人是什么人（身份叙事的底层驱动）
- **绑情感**：这个产品让你感受到什么（起源故事、具身的底层驱动）
- **绑价值观**：这个产品代表什么立场（未来叙事、稀缺性的底层驱动）

判断标准：如果去掉品牌名，叙事还能套在竞品上，说明意义绑定失败。
FAB映射：主要用 **Benefits** 完成价值观传递。

### 轴2. 问题-解决方案（负面情绪释放）
叙事的心理引擎：先建构问题制造负面情绪（焦虑/不安/不满），再通过解决方案释放负面情绪，完成情绪闭环。
- 问题建构 → 负面情绪积累 → 解决方案呈现 → 情绪释放 → 行动驱动
- 问题越精确、越切身，负面情绪越强，释放时的快感越大，转化越强

FAB映射：用 **Advantages + Benefits** 制造痛点并释放。

### 轴3. 认知节奏控制
控制受众在叙事中的注意力曲线。核心变量：
- **张力**：制造悬念/冲突/反直觉 → 注意力上升
- **释放**：给出答案/展示结果/确认情感 → 满足感
- **重置**：切换话题/转场/新信息 → 防止疲劳

节奏公式：`张力 → 张力加深 → 释放 → 重置 → 新张力`
短内容只做一个周期，长内容做多个周期。
FAB映射：用 **Features** 作为信息不对称的起点，逐步释放 Advantages/Benefits。

---

## 二、模态（叙事载体）

- **文本**：精确控制语义，适合理性说服
- **符号**：Logo、视觉锤、品牌色，适合高频重复认知
- **图像**：静态场景/产品展示，适合美学传递
- **视频**：动态叙事的主力载体
- **音频**：声音标识、播客深度内容、语音调性
- **具身**（跨视频+音频的特殊子类）：
  - 面孔：前语言信任（134ms内完成判断，窗口越短权重越大）
  - 表情：情感节奏（皱眉=张力，微笑=释放，惊讶=重置）
  - 姿态：关系姿态（仰拍=权威/aspirational，平拍=同侪，俯拍=亲近/服务）
  - 语言身份：背书资格（谁在说 > 说什么）

---

## 三、FAB（内容翻译器）

FAB不是独立层，而是贯穿轴与操作维度的翻译器——把产品事实翻译成叙事语言。

- **F - 属性特征**：产品是什么、有什么、用了什么
- **A - 差异优势**：和别人比好在哪
- **B - 益处**：对用户意味着什么

关键原则：**B在叙事框架中不是功能性的（省时间/省钱），而是身份性和情感性的（你成为谁/你感受到什么）。**

### FAB × 操作维度交叉（执行时必须过这张表）

| 操作维度 | F 怎么讲 | A 怎么讲 | B 怎么讲 |
|---------|---------|---------|---------|
| 稀缺性 | 限量工艺/材料/产地 | 别人没有这个 | 错过即永失 |
| 身份叙事 | 设计语言/美学体系 | 这代表某种品味 | 用它的人是这类人 |
| 问题-解决 | 技术参数/成分/机制 | 比现有方案好在哪 | 你的痛点消失了 |
| 起源故事 | 创始人的某个执念 | 因为执念所以不妥协 | 你买到的是一种信念 |
| 未来叙事 | 正在做的事/路线图 | 这件事只有我们在做 | 你在参与一个展开中的故事 |

## 四、操作维度（叙事策略）

### 4.1 稀缺性
- 稀缺=价值：限量、限时、限定渠道
- 错过=损失：损失厌恶驱动行动
- ⚠️ 失败模式：假稀缺被识破 → 品牌信任崩塌

### 4.2 身份叙事
- 我是谁？
- 我的审美品味是什么？
- 身份标签颗粒度细化（不是"年轻人"，是"在996和独立之间挣扎的28岁设计师"）
- ⚠️ 失败模式：标签太宽 → 谁都是等于谁都不是

### 4.3 问题-解决（执行层）
区别于轴2的心理机制，这里是具体执行：
- **问题建构**：从普遍性人群焦虑到精确场景性需求，选择什么问题、怎么呈现
- **方案呈现**：产品如何出场解决叙事制造出来的问题
- **场景选取**：在什么具体场景下触发问题感知
- ⚠️ 失败模式：问题建构过度 → 贩卖焦虑被反感

### 4.4 起源故事（向后的时间深度）
- 叙事为产品赋予时间深度
- 创始人使命："我觉得···不应该···"
- 老=可信，迁移至：他的理念=我认同的理念
- 消费者信任机制从权威向同侪者迁移
- ⚠️ 失败模式：过度包装 → 被扒皮后反噬

### 4.5 未来叙事（向前的时间牵引）
- "我们正在……"（进行感）
- "下一步是……"（方向感）
- "和我们一起……"（参与感）
- 作用：复购引擎、社群归属、持续关注的理由
- ⚠️ 失败模式：画饼不兑现 → 信任透支

## 五、叙事媒介适配（执行层）

### 原则
- 叙事永远要适配媒介的时间-注意力结构
- 单次叙事窗口被极端压缩，触达频次指数级增长
- 高频负责认知阶段
- 深度叙事负责决策阶段

### 窗口-内容适配
- **3秒**：只打B，具身面孔先行
- **15秒**：B→A，益处钩住，优势留住
- **60秒**：问题-解决完整弧，或起源故事片段
- **长内容**：F→A→B完整链，未来叙事展开

### 频次-深度协同
- 高频触达：统一符号+统一面孔+统一B点，种认知
- 深度内容：承接高频种下的认知，完成决策转化
- **关键约束：高频和深度必须是同一个叙事，不能各讲各的**

## 六、度量

每个维度的效果指标：

| 维度 | 核心指标 | 判断标准 |
|------|---------|---------|
| 意义绑定 | 品牌联想测试 | 用户自发描述中是否出现你预设的意义词 |
| 问题-解决方案 | 痛点共鸣率 | 评论/弹幕中出现"我也是""太真实了"等共鸣表达 |
| 认知节奏 | 完播率/阅读完成率 | 内容是否能hold住注意力到最后 |
| 稀缺性 | 首发转化率/售罄速度 | 稀缺叙事是否驱动即时行动 |
| 身份叙事 | UGC中的自我标签 | 用户是否用你的语言描述自己 |
| 问题-解决 | 搜索词变化 | 用户是否开始搜索你定义的问题 |
| 起源故事 | 主动传播率 | 故事是否被转述 |
| 未来叙事 | 复购率/社群活跃度 | 用户是否持续参与 |
| 具身 | 人物辨识度 | 遮住Logo后是否还能认出是你 |

## 使用流程

当收到一个产品叙事需求时，按以下顺序执行：

### Step 1：锁轴
- 轴1 意义绑定：方向是什么？（身份/情感/价值观）→ 输出一句话意义锚点
- 轴2 问题-解决方案：建构什么问题？释放什么负面情绪？
- 轴3 认知节奏：这个叙事的张力-释放节奏怎么设计？

### Step 2：选操作维度
问：当前阶段最需要哪1-2个操作维度？
引导问题：
- 用户知不知道自己有这个问题？（不知道 → 问题-解决）
- 品牌有没有可信的时间故事？（有 → 起源故事）
- 产品能不能成为某种身份的标志？（能 → 身份叙事）
- 供给是否真的有限？（是 → 稀缺性）
- 品牌有没有一个正在展开的方向？（有 → 未来叙事）

### Step 3：填FAB
在选定的操作维度下，逐一填写F、A、B
B必须是身份性或情感性的，不接受纯功能性表述

### Step 4：定模态和具身
问：用什么载体？需不需要具身？谁来具身？
具身人选的判断：面孔可信度 > 专业度 > 知名度

### Step 5：做媒介适配
根据投放渠道，按窗口-内容适配表裁剪内容
确保高频和深度素材是同一个叙事的不同切面

### Step 6：设度量
为选定的操作维度设定对应指标，定义"做到什么算成功"

---

## 输出格式

完成分析后，按以下格式输出叙事方案：

```
【产品】：___
【意义锚点】：一句话
【问题建构】：建构什么问题 → 释放什么情绪
【认知节奏】：张力-释放设计
【操作维度】：选定的1-2个
【FAB矩阵】：按选定维度填写的表格
【模态方案】：载体选择 + 具身策略
【媒介适配】：各渠道的内容裁剪方案
【度量指标】：核心指标 + 成功标准
```
